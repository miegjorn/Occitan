# Occitan Stack

A self-hosted AI agent infrastructure running on Matrix and Kubernetes.

The Occitan stack is a collection of persistent, memory-bearing agents that communicate over
Matrix rooms. Agents maintain continuity across sessions through a durable memory layer
(Farga), are coordinated by GitOps (ArgoCD), and run on Kubernetes. All production
components are written in **Rust**.

This repository is the public runtime layer: bootstrapper scripts, ArgoCD deployment
manifests, a Python mock image for local development, and generated documentation.

---

## Architecture

```
Matrix (Synapse homeserver)
    │
    ├── Charradissa ── Matrix ↔ agent bridge
    │       │
    │       └── routes messages to component agents by room
    │
    └── Agents
         ├── Guilhem  — org-level chronicler (persistent memory via Farga)
         └── [component agents — Gardian, Farga, Amassada, ...]

Kubernetes cluster (KinD locally, any k8s in production)
    ├── ArgoCD        — GitOps controller (this repo is the source of truth)
    ├── Synapse       — Matrix homeserver
    ├── PostgreSQL    — per-component databases
    ├── OpenBao       — secrets management
    └── Components
         ├── Gardian     — secret gateway          (Rust)
         ├── Farga       — durable memory / KV     (Rust)
         ├── Amassada    — context assembler        (Rust)
         ├── Charradissa — Matrix bridge            (Rust)
         ├── Cor         — core services            (Rust)
         └── Fondament   — foundation layer         (Rust)
```

## Quick Start

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/)
- [KinD](https://kind.sigs.k8s.io/docs/user/quick-start/#installation) v0.20+
- [kubectl](https://kubernetes.io/docs/tasks/tools/)
- [Helm](https://helm.sh/docs/intro/install/) v3+
- [ArgoCD CLI](https://argo-cd.readthedocs.io/en/stable/cli_installation/)

### Bootstrap

```bash
# 1. Spin up a KinD cluster with a local registry
export CLUSTER_NAME=occitan
export REGISTRY_PORT=5001
./bootstrap/kind-up.sh

# 2. Install ArgoCD and hand control to GitOps
export MATRIX_DOMAIN=matrix.example.com
export ARGOCD_REPO_URL=https://github.com/miegjorn/Occitan
./bootstrap/argocd-bootstrap.sh
```

This brings up all components pointing to the [mock echo-agent](./mock/echo-agent/) by default.

To run the real Rust components, override image values in your own `values-production.yaml`
(never committed to this repository).

## Design Principles

- **Memory over statefulness.** Agents are stateless processes; continuity lives in Farga.
- **Matrix as the nervous system.** All inter-agent communication goes through Matrix rooms.
- **GitOps all the way down.** No manual `kubectl apply` in production — everything through ArgoCD.
- **Rust for reliability.** Production components are written in Rust for correctness and performance.
- **Self-referential by design.** Agents generate the context that future agents are grown from.
